        Here use this part from

         

        REMOVING

        For your players to be able to remove the custom buildables they make you will need to do the following.

        Go into your custom variables again and find this line.

        dayz_allowedObjects =

        just below this line

        WG_OwnerRemove =[];

        and insert any of the class names of items you want in the custom build system.

        Go back into your fn_selfActions.sqf and find

        _isModular = _cursorTarget isKindOf "ModularItems";

        and change it to this

        _isModular = (_cursorTarget isKindOf "ModularItems") or ((typeOf _cursorTarget) in WG_OwnerRemove); you must remember to add your custom items to the "dayz_allowedObjects" and "DZE_maintainClasses" sections in your variables.sqf any object you add to "DZE_isRemovable" will be removable by anyone!

         

        // List of removable items that require crowbar
        DZE_isRemovable = ["Fence_corrugated_DZ","M240Nest_DZ","ParkBench_DZ","Plastic_Pole_EP1_DZ","FireBarrel_DZ","Scaffolding_DZ"];

         

        Remove the classnames from DZE_isRemovable that you dont want any player to be able to remove and add it to

        WG_OwnerRemove =[]; so that the player that builds it can remove it dont forget to make the edit to fn_selfactions

         

        _isModular = _cursorTarget isKindOf "ModularItems";

        and change it to this

        _isModular = (_cursorTarget isKindOf "ModularItems") or ((typeOf _cursorTarget) in WG_OwnerRemove);

         

        your variables might not have WG_OwnerRemove =[]; so just add it under dayz_allowedObjects line in variables.sqf

        yes this would work, problem is that you (as owner of the object) also are not able to remove them ^^

         

        a little better way (atleast in my opinion would be this:

         

        you need a custom variables.sqf and a custome remove.sqf (both from your dayz:_code.pbo)

         

        remove.sqf:

         

        search for:

        _objType = typeOf _obj;

        add right below:

        _isallowedremove = false; 

        search for:

        _isModular = _obj isKindOf "ModularItems";

        add right below

        _is_Removable = _objType in DZE_OwnerRemovable;

        now search for:

        _nameVehicle = getText(configFile >> "CfgVehicles" >> _objType >> "displayName");

        add right below

        if (!_isallowedremove && _is_Removable) exitWith {DZE_ActionInProgress = false; cutText [format["You are not allowed to remove this object (%1)",_nameVehicle], "PLAIN DOWN"]; }; //maybe change the text to your liking

        now we need to add every new variable to the private [] part at the top

        "_is_Removable","_isallowedremove", 

        done with remove.sqf

         

        variables.sqf:

        search for

        dayz_allowedObjects 

        add below:

        DZE_OwnerRemovable = ["Sandbag1_DZ","AnotherClassName","ClassnameObject"]; 

        add more classnames in there, which should not be allowed if you are not the owner.

         

        Safe and done 

         

        This way its only allowed to the owner and his friends to remove this objects ;)

        not tested!
