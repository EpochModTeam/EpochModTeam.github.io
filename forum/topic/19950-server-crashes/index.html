        I've had the same problem with my Arma2OAServer, and unfortunately, the causes of it are quite annoying. I haven't got any other cases with which to confirm this, but in my case, I ran ARMA2OASERVER with a vanilla (no mods or alterations) PBO which I ran on a server to make sure it worked without a hitch. I then ran this on my localÂ server, and would experience intermittent crashes with NTDLL.DLL being the case, and if you look further up in your RPT, you will see that the arma2 MALLOC is what is causing the error.

        Â 

        From what I saw, it doesn't matter what was in the script -- it was caused due to the fact that the files and registry entries used were corrupted and needed to be replaced, and in my case, the windows dlls themselves. NTDLL and MALLOC errors refer to the fact that the arma2oaserver is attempted to either access protected memory or the memory allocated to another program. Malloc, or "memory allocation" libraries like the one ARMA uses are used specifically to side-step this problem. When I saw this crashing due to memory allocation errors, I knew something was wrong.

        Â 

        To debug this, you need to:

        • If you're also having problems loading into ARMA2 in general, or have random crashes in ARMA2/A2OA, you may have some problems with your game files. Use steam to verify the games and then allow it to re-download and patch the ones which are causing problems.
        • If you are developing, look at any DLLs you are using and see if they are churning out tons of data and passing it to your script. If it is and you know how to debug DLLs, then do so.
        • If you have been having problems with other parts of windows, games, or applications crashing randomly with errors of a similar nature (i.e. "Memory cannot be 'read'" or "Memory allocation error"), then it's necessary to check if your RAM is physically faulty. To test this you can go to : http://windows.microsoft.com/en-us/windows7/diagnosing-memory-problems-on-your-computer, and follow the instructions. It may be that if your memory stick is faulty you will need to replace it.
        • If you are using other programs which are memory-intensive, then it may be that windows has ran out of available memory. Increasing virtual memory can alleviate this problem, however, I would strongly recommend that if you are peaking memory running the server with other stuff, it's time to either choose which process to run, or get two separate servers.
        • In my case, the cause of it is the DLLs in windows itself. You can try to use the sfc /scannnow command in the "Windows Key + R" box, which will force windows to check the integrity of its own files and replace those which are faulty.
        • For me, the only way that solved this isssue was to perform a full and complete format and re-install of windows.Â It is feasible that the exact cause of this can be resolved without formatting windows, however, I chose to do so because my PC was a mess, anyway. If you do not want to format, you can recreate the same effect by using an "enhanced uninstaller", like your Uninstaller or another similar product, and re-install ARMA2/A2OA. Make sure that when you re-install ARMA2OA, you delete the "@" mod folders inside your Steam common folder as these seem to be left intact by steam when the uninstall completes.
        • Your Uninstaller has worked for me before with similar issues, because it will remove all registry entries for a program on your computer, effectively making it as though you have never installed ARMA2 before. Surprisingly, uninstallers still leave a lot of crap behind. You could also look for registry cleaners and other products to get rid of any registry entries.
        • A combination of defragmenting your PC and running sfc /scannow on your drives may also prove a fix for this issue.

        If you do re-install windows and it fixes this issue, I would recommend using a program like CloneZilla to make a working copy of your Windows once everything you need is installed. Then if you need to format, you can just run filezilla and restore the disk image, and get back on windows far quicker.

        Â 

        Anyway, hope this helps. I found nothing on Google or even Bohemia's site for this problem, and thought it would be useful to provide advice on it. This motherfucker cost me 2 weeks of development time, at least. :angry:
