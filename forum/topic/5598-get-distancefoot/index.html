        I didn't try it myself and also I don't know which HiveExt you are using but with the official Epoch HiveExt from the current source (not the one in the server download) you should use CHILD:999 like this:

        _key = format ["CHILD:999:SELECT DistanceFoot FROM character_data WHERE PlayerUID=? AND Alive=1:%1", _uid];

        because it uses a prepared sql query, so you have to replace all values in teh query with "?" and set the values as extra parameters to the child call (delemiter should be ":" as always)

        escape of the string might not be needed so if that doesn't work try with PlayerUID='?'
