        it doesnt matter if i set it to local (the time the server hardware is set to) or custom with an offset. it still gives the client the time of when the server started, not the time when the client connected.

         

        Example:

        The Server Starts at 6:00 pm

        Client A logs in 5 min later, their watches read 6:00, its still day light, but the sun is going down.

        And hour has passed and the sun has set.

        Client B logs in, his watch reads 6:00, even though real time is 7pm, its still day light, but the sun is going down

        Client A is in complete darkness as the sun has gone down on his client, doesn't see Client B and is killed by Client B because its still day light for Client B.

        Client relogs and repawns and BAM, his watch reads 6:00, it is now 7:30pm server time, but his client is reading 6:00pm. Its daylight, but the sun is going down.

         

        This is broken, it was fixed in one of the DayZ updates (I am too lazy to go find which one) but that is for the full mod, and from what I understand, Epoch is built off DayZ Lite, not the full mod. So, the fix might just be missing. But, I am just a dumb server owner wrestling with server bugs, most of which are my own doing.
