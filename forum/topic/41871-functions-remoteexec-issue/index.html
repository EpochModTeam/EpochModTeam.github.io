        This message in the RPT is just a notice that a remoteExec call has been blocked.The reason you see it nowis that I removed the BIS MP framework overrides so we are usingnormal Arma 3 behavior.

        To suppress the PRT entry 'bis_fnc_execvm'could be allowed via remoteExec config and then simply add a emptyinitPlayerServer.sqfin the mission root. The battleye filter for remoteExec should still stop any other uses of'bis_fnc_execvm'. Otherwise it should only happen on each player connection so it could be left if the extra logging is not an issue.
