        So my qustions is. In what file do I find where they are on the map, or does it go after the trader skin, for exemple Air Plane Dealer = "Soldier_Sniper_PMC"?

         

        TO PLACE TRADERS ON THE MAP

         

        inside your server.pbo / missions folder / map folder your using so in your case it would be in DayZ_Epoch_17.Chernarus

        you have 2 files 1 is for vehicles random spawned on the map and the other is for your traders

         

        dynamic_vehicle.sqf

        mission.sqf

         

        Open mission.sqf this is a trader

         

        _unit_36 = objNull;
        if (true) then
        {
          _this = createAgent ["GUE_Villager4", [8387.1025, 2369.3113, 0.61760527], [], 0, "CAN_COLLIDE"];
          _unit_36 = _this;
          _this setDir 171.99826;
          _this setVehicleInit "this allowDammage false; this disableAI 'FSM'; this disableAI 'MOVE'; this disableAI 'AUTOTARGET'; this disableAI 'TARGET'; this setBehaviour 'CARELESS'; this forceSpeed 0;";
          _this setUnitAbility 0.60000002;
        _this allowDammage false; _this disableAI 'FSM'; _this disableAI 'MOVE'; _this disableAI 'AUTOTARGET'; _this disableAI 'TARGET'; _this setBehaviour 'CARELESS'; _this forceSpeed 0;_this enableSimulation false;};

         

        Where do I find the spawn point for vehicles that you buy, or does that go after just a helipad? So if you make a vehicle trader and place a helipad next him the vehicles will spawn there?

         

        Vehicles will spawn in the direction towards the closest helipad so place 1 near the vehicle trader to position vehicle spawn

         

        Can i combine different trader menus from different traders to make a new one? So I take the trader ID from two and make them to a new trader, will that work?

         

        its like this if you set a trader with a skin/classname like the 1 above GUE_Villager4 and you give him Menu TIDs of 558 and 557

        Boat Dealer Golove "GUE_Villager4" [["Boats Armed",558],["Boats Unarmed",557]],

         

        if you want to say place 4 boat dealers on the map with the same inventory or items/vehicles to trade by all means use the same trader skin/classname and same Menu TIDs

        Boat Dealer Golove "GUE_Villager4" [["Boats Armed",558],["Boats Unarmed",557]],

         

        if you want to place 4 boat dealers but want different menus you will have to use different skin/classnames for each trader

        note that you could still add the same Menu TIDs and add a few different Menu TIDS to customize each trader to have different menus

         

        this is the world space location of the trader [8387.1025, 2369.3113, 0.61760527] where trader will spawn on map

        this is the direction trader is facing _this setDir 171.99826;

         

        So if you wanted you could have a trader made up with many Menu TIDs that other traders are using on the map you would not need to set up new .hpp files!

         

         

        And what are the files that has to do with the traders? I know of server_traders.sqf ond all of the -hpp files. Is there any more.

         

        normal traders use sql tables in your database

        Cfgtraders use CfgServerTrader folder with .hpp files in your mission files

        either way both ways use

        Server_traders.sqf in your mission folder (to set trader Menu TIDs to trader)

        mission.sqf in your server.pbo / missions folder / map folder your using (Placement of trader on map)

        Mission.sqm in your mission files (class Sensors) what Corper said.

         

        For Cfgtraders

        cfgServerTrader.hpp is what calls the Category .hpp files

        each Category .hpp file has a TID and list of items with prices

        i told you in the first response if you want to mark the traders, with safezones and text. edit these "sensors" in the mission.sqm

        class Sensors
{
items=6;      //Total number of sensors, Remember it starts with zero
class Item0   // sensor starts 0 then 1,2 and so on
{
position[]={x,z,y}; //positon of the sensor, location from the editor u get x,y,z, remmber to change positions
activationBy="WEST";
repeating=1;
interruptable=1;
age="UNKNOWN";
name="TraderCityName"; //make a name for the trade city i think its got to be unique
expCond="(player distance TraderCityName) < 50;"; //name got to match  name above and 50 is the radius size of ur safezone
expActiv="[""trader city Friendly"",true,""enter""] spawn player_traderCity;"; //message when entering
expDesactiv="[""trader city Friendly"",true,""leave""] spawn player_traderCity;";  //message when leaving
class Effects
{
};
