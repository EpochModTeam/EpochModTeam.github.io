        I had RConPassword in expansion/battleye/beserver_active_xxxxxxxx.cfg and password in config.php both set to "zombieLove" (passwords changed to protect the stupid) and passwordAdmin in cfgdayz/server.cfg set to "zombiieLove".

        Using "zombiieLove" in config.php (technically the correct password) caused a "Login Failed!" message because it didn't match battleye's RConPassword.

        Using "zombieLove" (technically the wrong password) passed the wrong password check in the php code but hung the connection with no error message.

        Â 

        Correcting passwordAdmin in cfgdayz/server.cfg to "zombieLove" has resolved the issue totally.

        I hope I've explained in an understandable way, it's one of the those subtle "logical errors" that doesn't cause an error but stops the code from working properly.

        Hm, the original script did always work well for me.

        Â 

        Just called them via cron like this:

        00 2,6,10,14,18,22 * * * php5 /home/dayzepoch/Restart/60min.php >/dev/null 2>&1

        Â 

        Just check if you can call php or php5 from the shell directly with the scripts, then it also works from cron. As Trollspace said, always use absolute paths (to the scripts, not essentially to the application).

        It' s a bit annoying that there is a script for each time...but after setting this up once you never care about it.

        Â 

        The only problem is that cron execute these at exactly the time you define...but maybe the server has been restartet in between. So until the next regular restart the time you may show in a custom debug monitor ('Next restart in xxx minutes' is not the same as the cron script message shows.
