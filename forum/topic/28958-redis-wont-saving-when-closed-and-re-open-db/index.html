        Someone more knowledgeable than me may want to chime in here, but my understanding of the way it works is it only writes to disk when there are 'X' number of new records to be saved. Â Check the conf file in the db folder and you will see there are some settings in there that can be changed. Â 

        save 900 1
save 300 10
save 60 1000

        So basically that's telling Redis to save every 900 seconds if there is 1 record to update, every 300 seconds if there are 10 records to update, and 60 seconds if there are 1000 records to update. Â So what's happening is your players are moving around, changing inventory items, doing trader transactions, and then the server restarts, but the transaction hasn't been written to disk yet. Â So the next time the server restarts, those transactions can't be pulled back into game, because according to Redis they never happened. Â 

        Â 

        What I did to combat this is to change the settings to as follows:

        save 900 1
save 300 1
save 5 1

        This way the server is saving every 5 seconds, regardless of how many records there are to update. Â How effective this will actually be has yet to be seen. Â But it makes sense that it should work. Â
