        The question here I guess is it not if you can maintain the objects if they do not have damage. Maintaining just means set back the damage to 0. o as they have 0 damage for you...no need to maintain them.

        The question is how you remove the objects if there is no damage as normally they get removed if damage = 1 or something like this.

         

        But I think you can find the place in dayz_code/actions/maintain_area.sqf (make it custom) where the objects get filtered for how many damage they've got:

        //filter to only those that have 10% damage
_objects_filtered = [];
{
    if (damage _x >= DZE_DamageBeforeMaint) then {
        _objects_filtered set [count _objects_filtered, _x];
   };
} forEach _objects;
_objects = _objects_filtered;

        ...and you do not need the part with the if...just take all the objects and get rid of any filtering here. That would allow you to maintain at any time. If it's only about to maintain just to get avoid getting more and more objects on the server I would also delete the costs here.

         

        Comment out all of this if you want that:

        case (_count <= 10):  {_requirements = [["ItemGoldBar10oz",1]]};
	case (_count <= 20):  {_requirements = [["ItemGoldBar10oz",2]]};
	case (_count <= 35):  {_requirements = [["ItemGoldBar10oz",3]]};
	case (_count <= 50):  {_requirements = [["ItemGoldBar10oz",4]]};
	case (_count <= 75):  {_requirements = [["ItemGoldBar10oz",6]]};
	case (_count <= 100): {_requirements = [["ItemBriefcase100oz",1]]};
	case (_count <= 175): {_requirements = [["ItemBriefcase100oz",2]]};
	case (_count <= 250): {_requirements = [["ItemBriefcase100oz",3]]};
	case (_count <= 325): {_requirements = [["ItemBriefcase100oz",4]]};
	case (_count <= 400): {_requirements = [["ItemBriefcase100oz",5]]};
	case (_count <= 475): {_requirements = [["ItemBriefcase100oz",6]]};
	case (_count <= 550): {_requirements = [["ItemBriefcase100oz",7]]};
	case (_count <= 625): {_requirements = [["ItemBriefcase100oz",8]]};
	case (_count > 700):  {_requirements = [["ItemBriefcase100oz",9]]};

        Then you would need an database event to get rid of objects which have not been updated since x days or something (table object_data, column last_updated to check in the where clause).

        There are examples in the forums how to create such an event I guess.

         

        I did not test it and I do not know (as I don't have the original hive-update calls in the linux server port) if the object is updated if damage is 0 but that is the way I would go to check this out.
