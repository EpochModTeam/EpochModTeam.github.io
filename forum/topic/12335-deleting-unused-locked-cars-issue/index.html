        Hi,

        We have a panthera Epoch server hosted by dayz.st and we want to do some cleanup on the map.

        The server restarts every 6 hours and we have many many cars that are locked and haven't been updated for more than 20 days.

         

        Following the pattern of the sql file that creates events and comes with the epoch server I edited a few and created and event

        that will unlock all the locked vehicles that have not been updated in the past 15 days and are older than 30 days.

         

        here is the code:

        DROP EVENT IF EXISTS `UnlockOldUnusedVehicles`;
DELIMITER ;;
CREATE EVENT `UnlockOldUnusedVehicles` ON SCHEDULE EVERY 6 HOUR DO
UPDATE 
   `Object_DATA`   
SET
   `Object_DATA`.`CharacterID` = 0
WHERE
   `Object_DATA`.`CharacterID` <> 0
   AND `Object_DATA`.`CharacterID` <= 12500
   AND `Object_DATA`.`Classname` NOT LIKE 'Tent%'
   AND `Object_DATA`.`Classname` NOT LIKE '%Locked'
   AND `Object_DATA`.`Classname` NOT LIKE 'Land%'
   AND `Object_DATA`.`Classname` NOT LIKE 'Cinder%'
   AND `Object_DATA`.`Classname` NOT LIKE 'Wood%'
   AND `Object_DATA`.`Classname` NOT LIKE 'Metal%'
   AND `Object_DATA`.`Classname` NOT LIKE '%Storage%'
   AND `Object_DATA`.`Classname` NOT IN ('OutHouse_DZ', 'GunRack_DZ', 'WorkBench_DZ', 
				'Sandbag1_DZ', 'FireBarrel_DZ', 'DesertCamoNet_DZ', 'StickFence_DZ', 'LightPole_DZ',
                'DeerStand_DZ', 'ForestLargeCamoNet_DZ', 'Plastic_Pole_EP1_DZ', 'Hedgehog_DZ', 'FuelPump_DZ',
                'Fort_RazorWire', 'SandNest_DZ', 'ForestCamoNet_DZ', 'Fence_corrugated_DZ', 'CanvasHut_DZ', 'Generator_DZ')
   AND `LastUpdated` < DATE_SUB(CURRENT_TIMESTAMP, INTERVAL 24 DAY) 
   AND `Datestamp` < DATE_SUB(CURRENT_TIMESTAMP, INTERVAL 42 DAY)
 ;;
DELIMITER ;

        Here is my dilemma:

         

        It looks that the event does not affect the database when the server is running, and after the server restarts those vehicles are shown in the database that

        have a "CharacterID = 0" but they are still locked. so in game they are shown with "Vehicle Locked"

         

        As far as I learned is that if you have a car and you leave it unlocked, after restart the server locks it back, so i don't know how this event would achieve what is supposed to?

        First of all it does not affect the database right away and then after the server restarts the servers tells what to do to the vehicles that were with key and unlocked.

         

        Please someone clarify some of these issues to me.

         

        On another note, I deleted one row in the "Object_DATA" for a vehicle using the PhPMyAdmin and using the button delete and not an sql command and the vehicle in that row is not to be found anymore in the "Object_DATA", however, after I restart the server and log back in, that vehicle that was at the specified row is still in the game but not in my "Object_DATA".

         

        How is that possible?

        This might help... link... apparently it checks if a vehicle has a key somewhere on the server and if not it will either unlock it or delete it... im not really sure... and I have not tested it.

         

        I also found this on google:

        UPDATE object_data SET CharacterID = 0 WHERE LastUpdated < DATE_SUB(CURRENT_TIMESTAMP, INTERVAL 14 DAY) AND Datestamp < DATE_SUB(CURRENT_TIMESTAMP, INTERVAL 14 DAY) AND CharacterID > 0 AND Classname NOT IN ('VaultStorage','LockboxStorage','VaultStorageLocked','LockboxStorageLocked','WoodShack_DZ','StorageShed_DZ','TentStorageDomed','TentStorageDomed2','TentStorage') AND Inventory <> '[]' AND Inventory IS NOT NULL

         or alternatively:

        DROP EVENT IF EXISTS unlockAbandonedVehicles;
CREATE EVENT unlockAbandonedVehicles
ON SCHEDULE EVERY 1 DAY
COMMENT 'Unlocks vehicles that have been abandoned'
DO
UPDATE `Object_DATA` SET `CharacterID` = 0 WHERE `LastUpdated` < DATE_SUB(CURRENT_TIMESTAMP, INTERVAL 4 DAY) AND `Datestamp` < DATE_SUB(CURRENT_TIMESTAMP, INTERVAL 10 DAY) AND `CharacterID` > 0 AND `Classname` NOT IN ('VaultStorage','LockboxStorage','VaultStorageLocked','LockboxStorageLocked','WoodShack_DZ','StorageShed_DZ','TentStorageDomed','TentStorageDomed2','TentStorage')  AND `Inventory` <> '[]' AND `Inventory` IS NOT NULL;

        I have not tested these but I thought it might help...
