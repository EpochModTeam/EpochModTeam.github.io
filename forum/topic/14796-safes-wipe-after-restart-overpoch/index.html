        Nothing safe related is a half truth, your error is not safe related but since the error possibly breaks the script flow when the error is thrown it can have odd effects.

         

        In your server log you have these lines.

        13:19:55 Error in expression <s select _index;
_itemType = _itemTypes select _index;
_veh addMagazineCargoGlob>
13:19:55   Error position: <select _index;
_veh addMagazineCargoGlob>
13:19:55   Error Zero divisor
13:19:55 File z\addons\dayz_server\init\server_functions.sqf, line 370

        Find this section around line 370 a few lines higher than where this appears you will see _index declared, it then appears to be recycled in the code below, not programming best practice.

        _index = floor(random _cntWeights);
_index = _weights select _index;
_itemType = _itemTypes select _index;

        The fix is to change these three lines to read.

        _index1 = floor(random _cntWeights);
_index2 = _weights select _index1;
_itemType = _itemTypes select _index2;

        The fix was posted in github as a fix for supply_drops.sqf and crash_spawner.sqf being broken due to the recycling of those variables.
