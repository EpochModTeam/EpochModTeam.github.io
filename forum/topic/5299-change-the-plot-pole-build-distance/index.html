        This is what I did to increase the plot pole radius  I can now build larger base without multiple plot poles..
         
        unpack Dayz_mission.pbo
        unpack dayz_code.pbo
         
        copy player_build.sqf from dayz_code.pbo to your mission.pbo \custom folder
         
        find in player_build.sqf
         
        _distance = 30;
        _needText = localize "str_epoch_player_246";
         
         
        change the 30 to whatever distance you want
         
        The pole to pole distance is here
         
        if(_isPole) then {
        _distance = 60;
        };
         
        make sure your calling your compiles in the init.sqf
         
        init.sqf
         
        call compile preprocessFileLineNumbers "compiles.sqf";   //Compile regular functions
         
        change in compiles.sqf
         
        player_build = compile preprocessFileLineNumbers "\z\addons\dayz_code\actions\player_build.sqf";
         
        to 
         
        player_build = compile preprocessFileLineNumbers "custom\player_build.sqf";
         
        repack and upload

        Im a noob too,  sorry my instructions are verry bad.

        I am running epoch tavi with dayz.st and in my mission pbo root there is compiles.sqf.  I Dont think I had to add this it was already there. if you dont have it, you could create a new one or better yet, copy the original out of your dayz_code.pbo

        if you copy it from dayz_code, place it in your mission root. just find the line in compiles.sqf that calls player_build.sqf

        line 102    player_build = compile preprocessFileLineNumbers "\z\addons\dayz_code\actions\player_build.sqf"; 

         

        and then change the path to the folder you placed it. I have placed player_build.sqf in my custom folder

         

        player_build = compile preprocessFileLineNumbers "custom\player_build.sqf";

         

        calling "compiles.sqf"  is already in the init  mine is called at line 67

        if its not there with the rest  add it 

         

        init.sqf

         

        //Load in compiled functions
        call compile preprocessFileLineNumbers "\z\addons\dayz_code\init\variables.sqf"; //Initilize the Variables (IMPORTANT: Must happen very early)
        progressLoadingScreen 0.1;
        call compile preprocessFileLineNumbers "\z\addons\dayz_code\init\publicEH.sqf"; //Initilize the publicVariable event handlers
        progressLoadingScreen 0.2;
        call compile preprocessFileLineNumbers "\z\addons\dayz_code\medical\setup_functions_med.sqf"; //Functions used by CLIENT for medical
        progressLoadingScreen 0.4;
        call compile preprocessFileLineNumbers "compiles.sqf"; //Compile regular functions
        progressLoadingScreen 0.5;
        call compile preprocessFileLineNumbers "server_traders.sqf"; //Compile trader configs
        progressLoadingScreen 1.0;
