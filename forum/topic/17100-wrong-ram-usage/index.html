I have tested with "tbb3malloc_bi" and "tbb4malloc_bi" and having the startparameter "-maxMem=2047" (also tried -maxMem=2048).         I doubt that there's anything to worry about.  The discrepancy likely comes from the fact that the game is actually using ~800MB, but has also "reserved" an additional ~800MB or so in case it is needed.  The additional memory isn't actually being used "yet" but can be called upon by the server application if necessary.  This is done to prevent other Windows applications from using the reserved memory for their own purposes and insures that there is enough memory free for the server app to use as needed.  The only time I would be worried is if you start having crashes as a result of excessive memory usage, which could indicate a memory leak or faulty memory stick on the server box.
