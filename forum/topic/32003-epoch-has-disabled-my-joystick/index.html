        Hey guys! I've been playing Epoch nonstop for the past few weeks. However, when our server installed the newly released update a couple days ago, my joystick stopped being recognized by Arma. This was the only change in settings and config between things working and things not working. Now whenever I get into a vehicle my mouse is locked as input for vehicle instead of freelook, and my joystick won't function at all. I disabled the Epoch mod on my own game and the joystick worked fine. When I re-enabled the mod the joystick stopped working, even outside of the server. This is quite perplexing since obviously I didn't make any changes to my clientside files.

         

        Has anybdoy run in to something like this before? I've switched ports, turned off USB power saving, etc. and I have confirmed multiple times that enabling Epoch mod is what causes the trouble. JSRS, AIA, CBA, etc. all work fine.

         

        To be clear, my joystick appears int he Configure Controller menu only when Epoch isn't enabled. With Epoch off the joystick shows up properly in the Configure Controllers menu, and all Axis, buttons, etc. work like they've always worked. With Epoch enabled, it disappears from the Configure Controllers menu and no buttons or axis work. I tried verifying local content, running Arma without the additional parameters (you CPU count, memcount, etc.), running Arma from the Launcher instead of directly from Steam, running from an independent shortcut, etc. but all to no avail.

        I'm not sure if this is relevant to your situation.

        But there is an issue with some Saitek joysticks and Arma 3, X52 and X55 are two that I know of. 

         

        I am able to configure my X52 in the menu just fine,

        And I can use the stick in game, however if I bring up the options menu to adjust controls while in game. 

         

        The joystick won't display and will sometimes stop working after that point

        Until I restart Arma 3, This has happened to me in custom co-op missions using RHS and ALiVE.

        I've had it happen in KoTH, Epoch, Even in the editor while making mad edits for Bornholm.

         

        As mentioned above, if the server's using infi, his config has an option to disable custom keybinds, and i'm pretty sure it's set true by default. that would also mess it up for you.
