        When it comes to side missions and other epoch events, I find that it's a good idea to give a 10 minute settling in period for my players at server start and a 30 minute cool down period before the restart.  The following code assumes you're restarting every 180minutes (3 hours).  It first checks that the time to restart isn't more than 170minutes,  when it drops below 170 then 10 minutes have passed and it's ok to run the script.  It also checks that the server isn't less than 20 minutes away from a restart, if it is then the script exits.  This has made my missions and epoch events feel more professional in a timely fashion.

         

        The last part in particular is very useful because it stop missions starting 10, 5 or even 1 minute before server restart.  Which just irritates me personally :)
         

        if ((180-(round(serverTime/60))) > 170 ) exitWith { // Give people a chance to log in first! 
 };
if ((180-(round(serverTime/60))) < 20 )	exitWith { // No point running missions 20 minutes before reboot 
 };

        Note: The 180 is how long my server runs before a restart. The rest of that code is doing something with the time that I don't fully understand yet, but if anyone wants to help expand my knowledge please do :)

        Hope that helps someone as much as it did me :)
