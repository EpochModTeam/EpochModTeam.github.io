        I've begun having an issue with my Epoch Sauerland server as of the 1041 update.  I have the server set to automatically restart via BEC scheduler and a restart batch file.  This has always worked fine until now.  The problem is that every couple days or so the server refuses to shut down.  The console window disappears, but if you check Windows task manager it is still running in the background.  So to get it to shut down I have to force it to do so in task manager.  Even then it takes nearly 5 minutes for it to stop.  Consequently, when the new server restarts as intended via the scheduler, it does so on the wrong port, since the other server running in the background still has the original port occupied.  Which then means that people cannot connect to it, nor can the next instance of BEC, and so the next scheduled server restart never occurs.  This hasn't been an issue until now.  We've been running an Epoch server for well over a year with no problems like this.  

         

        I don't know that this is anything specifically related to Epoch or if it's an Arma problem, but it's really got me scratching my head.  I've got a few ideas to try but I wanted to throw up something here and see if anyone had any thoughts.  Thanks in advance!
