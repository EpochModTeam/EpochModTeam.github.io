        Hello,

         

        we run several instances of epoch on our server. Since many other applications require us to run a debian OS, we use wine to run Dayz Epoch.

         

        Wine does NOT support The Microsoft Visual C++ 2013 Redistributable Package. Not even the 2012 version is in any way a decent build.

         

        With this dependency you actively disable any linux based server to run Dayz Epoch.

         

        Will you revert this to the 2010 version, which runs perfectly under any operating system?

         

        Best regards,

         

        Doolittle.

        I ran into the same issue but the fix is easy:

         

        If you start the server without the redist package wine will tell you which dlls are missing:

        • MSVCP120.dll
        • MSVCR120.dll

        Just copy the missing 32-bit dlls(either from your windows or download them from the web) into your arma-root and it will run.

         

        GL&HF!
