        ah, check the prefix. Naming of PBO does not matter, as long as prefix is correct as they are loaded in automatically. Also for server files only keep 1 pbo in the directory.

         

        prefix should be z\addons\dayz_server

        you can see prefix using tools like PBOManager, open it up and press cog wheel to verify, this is important. Also make sure its called prefix on left field and NOT pboprefix. If it's pboprefix, you might want to consider changing tool you use to PBO files.

         

        Also make sure that in server launch params your @Server mod is loaded last. This is because MPMissions init.sqf is loaded first, then by default client side \z\Addons\dayz_code\... is loaded, when it gets to compiles file, then server side is only being loaded... you can see first file to be loaded bottom of compiles file.

         

        That's pretty much the sequence.
