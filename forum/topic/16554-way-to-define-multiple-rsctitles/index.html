        Hey guys

         

        Just a quick query about rsctitles in the description.ext.

        I have been looking into how this defined the UI ingame.

        My current issue is that i run multiple mods that modify and use this title, most of the time i will add a .h file and due to the rsctitle being defined within it it fries the server.

        So my work around has been to pile the file into the description.ext, which is not cool and something i dont want.

        Is there a way to tell say, btc_HUD.h file to define its objects within the rsc titles of the description.ext ?

        I have tested a few things out but to no avail.

        Any handy pointers woulf be cool :)

         

        Cheers

        put the rsctitles in description and include everything that needs to be in there INSIDE the "{  }"

         

        delete all rsctitles references in the includes you do.

         

         

        You can only overwrite a class once ( from the default dayz config.bin)

        class RscTitles {
#include "dzgm\icons.hpp"
#include "addons\playerhud\hud.hpp"

// and so on
}

        It's simple object orientated programming.
