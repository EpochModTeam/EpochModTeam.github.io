        After reading the documentation here there are a few minor tweaks that are needed.

         

        On line 209 in server_cleanup.FSM:

        class general_cleanup
    {
      name = "general_cleanup";
      init = /*%FSM<STATEINIT""">*/"//diag_log ""CLEANUP: Starting loop for next task"";" \n
       "" \n
       "//Clean groups" \n
       "{" \n
       "	// diag_log (""CLEANUP: CHECKING GROUP WITH "" + str(count units _x) + "" UNITS"");" \n
       "	if (count units _x==0) then {" \n
       "		deleteGroup _x;" \n
       "		// diag_log (""CLEANUP: DELETING A GROUP"");" \n
       "	};" \n
       "} forEach allGroups;"/*%FSM</STATEINIT""">*/;
      precondition = /*%FSM<STATEPRECONDITION""">*/""/*%FSM</STATEPRECONDITION""">*/;

        The line that sarge refers to is:

           "    if ((count units _x==0) && !(_x getVariable[""SAR_protect"",false])) then {" \n
   "        diag_log (""CLEANUP: DELETING A GROUP: "" + str(_x));" \n

        The magic here is this variable:

        (count units _x==0)

        Change line 215 (1.0.3.1) from:

        if (count units _x==0) then {" \n

        To:

        if ((count units _x==0) && !(_x getVariable[""SAR_protect"",false])) then {" \n

        Next we just have to change the cleanup for hackers line (Which is slightly different in epoch). Change this:

        if(vehicle _x != _x && !(vehicle _x in PVDZE_serverObjectMonitor) && (isPlayer _x)  && !((typeOf vehicle _x) in DZE_safeVehicle)) then {" \n

        to:

        if(vehicle _x != _x && !(vehicle _x in PVDZE_serverObjectMonitor) && (isPlayer _x) && (vehicle _x getVariable [""Sarge"",0] != 1) && !((typeOf vehicle _x) in DZE_safeVehicle)) then {" \n

        Thats about it for the different code. The BE filters you will have to do yourself (As this isn't my area of expertise).

         

        Not sure if there is anything in SARGEAI's codebase that will break when using epoch's dayz_code. Just test it out and see what errors get logged in the RPT.
