        The only part of the pubvar file that gets overwritten is the random variables assigned to it by the server on each restart.  The rest of the file remains untouched.  So if you have all of the correct exceptions in your BE filters then that part should be good and the kicks will stop.

         

        As for the bans, if you add anything to the server that adds functions (like those that show up on your scroll wheel as part of R3F) you will have to disable addon checking in your EpochAH settings.  Once they open it up and allow us to edit the AH properties we can add the menu actions and it will be fine.  But for now this will be the closest thing to a solution as we can get.

        Honestly if you're needing to add stuff for R3F to the pubvar file I would do it on a new line.  Setting the other functions in there to 1 or 2 is opening yourself up to a lot of security risk.  Just add it to a new line and then play with it from there.  Add it to the "original" file and then when the random variables get copied in then your additions should go with it.
