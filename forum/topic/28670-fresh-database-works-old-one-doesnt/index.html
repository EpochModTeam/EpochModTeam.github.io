        Im guessing you are or have the ability to make sql backup (export) from phpmyadmin. if so, use advanced feature to export all but object data into an sql.

         

        Then export ONLY the fresh object data into another sql.  import both into a new db and try from there. Then you can add stuff to the object data little by little until you find the real issue.

         

        Should also find some clues in the logs/rpt. Maybe it will tell you what object is causing the issue and you can just remove the object its rejecting then restart the server...
