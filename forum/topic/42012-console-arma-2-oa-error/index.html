        ntdll.dll is an essential windows component. In the error it is the Faulting Module, essentially the library that is reporting the error. That said, you shouldn't have that DLL in the arma server folder, only somewhere in Windows.

        There could be untold reasons why it is crashing, check the mod has been fully downloaded, make sure any DDLs in the arma server folder(s) have been unblocked.

        Then check your .rpt file to see if the server is crashing on anything specific as it loads, it is possible to crash the server via code, I have done it, accidentally of course :)

         

        EDIT: You have installed these ? :

        5.Visual C++ Redistributable Packages for Visual Studio 2013, http://www.microsoft.com/en-us/download/details.aspx?id=40784 (x86 is for 32 Bit.)
        6.Visual C++ Redistributable Packages for Visual Studio 2010, http://www.microsoft.com/en-us/download/details.aspx?id=5555
