        **edited edit**

        Ok, just clear this issue up, I discovered my problem. I had included asm.pbo in the server files (only to remove @asm from the command line). Removing asm.pbo and running it with @asm on the server command line cured the HC kick problem. I guess because the HC and the server are pulling from the same files, the unsigned asm.pbo the client loaded got it kicked.

        I'll do a work around later, but at least that problem was solved.
